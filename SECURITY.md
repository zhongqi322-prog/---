# 安全与敏感信息报告

## 报告方式

发现安全漏洞、隐私泄露、敏感信息暴露或高风险内容时，请不要在公开 Issue 中粘贴敏感数据。

请通过 GitHub 仓库所有者可用的私密联系方式报告，并提供：

- 问题描述
- 影响范围
- 复现步骤
- 建议修复方式

## 敏感信息范围

- 用户出生日期、出生地点和出生时间
- 用户住址、户型图和住宅照片
- 手相照片及其他用户上传图片
- 支付、订单或账户信息
- API Key、访问令牌和环境变量
- 未公开的古籍扫描资料或版权证明

## 项目安全原则

- 第一版不接真实支付或真实 AI。
- 用户隐私数据遵循最小收集原则。
- `.env`、密钥、令牌和真实用户数据不得提交到 GitHub。
- mock 数据不得使用可识别的真实个人信息。
- 高风险内容不得替代医疗、法律、投资或心理健康专业建议。
