# 古籍版权与版本确认表

## 1. 填写目的

本表用于确认当前已进入 `02_古籍录入/raw/` 的古籍 Markdown 是否可以用于本项目。

确认前，所有结构化片段继续保持：

- `copyright_status`: `待确认`
- `review_status`: `待审核`

确认后，Codex 才能按你的填写结果更新结构化数据状态。

## 2. 填写说明

请你主要填写以下列：

- `来源`：文件从哪里来。
- `版本`：对应哪个底本或整理版本。
- `可用范围`：允许本项目用到什么程度。
- `是否可用于正式产品`：填 `是` / `否` / `待确认`。
- `备注`：有不确定或需要替换来源的地方写在这里。

建议填写选项：

### 来源

- 自己录入
- 自己校对整理
- 维基文库
- 公版古籍库
- 古籍影印 OCR
- 网络公开整理
- 付费平台
- 现代出版社整理
- AI 整理生成
- 不确定

### 版本

- 四库全书本
- 维基文库本
- 中华典藏本
- 公版书整理本
- 网络整理本
- 现代注释本
- 影印 OCR 本
- 自校本
- 不确定

### 可用范围

- 仅内部整理
- 可用于 mock 原型
- 可用于公开网站展示
- 可用于商业付费报告
- 不可使用
- 待确认

## 4. 判断规则

可进入正式产品的最低建议条件：

1. 来源清楚。
2. 版本清楚。
3. 不属于现代出版社、付费平台或不明授权网站的受限整理文本。
4. 你明确允许用于公开网站展示或商业付费报告。

如果来源或版本不确定，建议保持：

- `可用范围`: `仅内部整理` 或 `可用于 mock 原型`
- `是否可用于正式产品`: `待确认`

## 5. Codex 后续处理规则

你填写本表后，Codex 可执行：

1. 根据 `是否可用于正式产品 = 是` 的书目，更新对应条目的 `copyright_status`。
2. 对仍为 `待确认` 的书目，保留当前状态，不进入正式产品引用。
3. 对 `不可使用` 的书目，将相关结构化条目标记为 `退回` 或从正式候选中移除。
4. 不根据未填写内容自动推断授权。
