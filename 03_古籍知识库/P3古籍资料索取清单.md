# P3古籍资料索取清单

## 1. 目的

固化 P2 审核建议优化项：P3 开始时必须向用户索取真实古籍资料，不得在前端 mock 数据中补写古籍原文。

## 2. P3 开始前必须向用户确认

请用户提供或确认以下内容：

1. 古籍书名
2. 版本来源
3. 是否确认可用于本项目
4. 原文片段
5. 对应章节或卷目
6. 用户希望优先结构化的主题
7. 是否已有 PDF、Word、Excel、CSV 或纯文本格式

## 3. 支持的资料格式

- PDF：使用 `pdf` Skill 处理。
- Word / 文档：使用 `documents` Skill 处理。
- Excel / CSV：使用 `spreadsheets` Skill 处理。
- Markdown / TXT：使用本地文件读取和结构化规则处理。

## 4. 首批建议索取方向

按 MVP 功能优先级，建议首批索取：

1. 八字命理相关片段
2. 紫微斗数相关片段
3. 风水理气 / 峦头相关片段
4. 易经问事相关片段
5. 周公解梦相关片段
6. 相术相关片段

## 5. 禁止事项

P3 阶段禁止：

1. 编造古籍原文。
2. 编造书名、章节、版本。
3. 把未经确认的内容写成正式出处。
4. 直接把前端 mock 占位当作真实古籍内容。
5. 输出绝对化预测或恐吓性解释。

## 6. 入库前检查

每条古籍片段入库前必须包含：

- 书名
- 版本或来源说明
- 原文片段
- 白话解释
- 关键词
- 适用场景
- 风险提醒
- 是否已审核
