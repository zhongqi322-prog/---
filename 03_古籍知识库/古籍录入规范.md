# 古籍录入规范

## 目标

将合法可用的古籍资料整理为可追溯、可审核、可被产品引用的知识片段。禁止用 AI 编造原文、出处或解释。

## 三层结构

1. 原始资料：`02_古籍录入/raw/书名/`
2. 清洗文本：`02_古籍录入/cleaned/书名.md`
3. 结构化知识：`data/classics/模块.json` 与 `03_古籍知识库/` 下的主题文档

## 录入前检查

- 确认书名、作者、年代和具体版本。
- 记录资料来源、获取日期和版权状态。
- 古籍原作进入公有领域，不代表现代整理本、注释本或扫描文件可自由使用。
- 版权状态未确认的资料不得进入正式产品知识库。

## 每条知识片段字段

| 字段 | 必填 | 说明 |
|---|---|---|
| `id` | 是 | 稳定唯一标识 |
| `book_title` | 是 | 书名 |
| `edition` | 是 | 使用版本 |
| `chapter` | 是 | 卷、篇或章节 |
| `source_text` | 是 | 保留原文 |
| `plain_explanation` | 是 | 克制的白话解释 |
| `keywords` | 是 | 关键词 |
| `module` | 是 | 八字、紫微、风水、易经、解梦、相术或其他 |
| `scenarios` | 是 | 适用场景 |
| `source_reference` | 是 | 可定位出处 |
| `copyright_status` | 是 | 待确认、可使用或不可使用 |
| `review_status` | 是 | 待审核、已审核或退回 |
| `uncertainty_notes` | 否 | 争议与不确定性 |

## 处理要求

- 保留原文，不擅自改写原意。
- 白话解释与原文分开。
- 不同流派观点分别标注。
- 不确定内容标记为“待确认”。
- 不做绝对化预测。
- 每条知识片段必须能够定位到具体出处。

## 验收

- JSON 或 Markdown 格式正确。
- 每条片段都有出处、版权状态和审核状态。
- 内容不影响已有功能。
- 更新任务看板与开发日志。
