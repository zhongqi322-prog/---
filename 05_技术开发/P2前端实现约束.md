# P2前端实现约束

## 1. 技术栈确认

P2 可点击原型阶段采用以下前端技术栈：

- Next.js
- React
- TypeScript
- Tailwind CSS

本阶段目标是完成可点击网站原型，不引入复杂后端架构。

## 2. P2 实现优先级

P2 前端开发 Agent 应按以下优先级实现：

1. 项目脚手架：Next.js + React + TypeScript + Tailwind CSS。
2. 统一页面壳：顶部导航、移动端布局、底部风险说明入口。
3. 首页：今日黄历卡片、核心功能入口、付费报告示例、风险说明。
4. 通用功能入口卡片：覆盖全部 MVP 功能，不得删减功能。
5. 通用表单组件：文本、日期、时间、选择项、图片 mock 上传。
6. 通用报告组件：摘要、详细分析、古籍出处占位、风险提醒。
7. 古籍出处占位组件：只展示“待 P3 接入用户提供古籍内容”，不得编造原文。
8. 风险提醒组件：所有报告页固定展示。
9. mock 支付弹窗：不产生真实扣款。
10. localStorage 记录：保存最近 mock 报告摘要和解锁状态。

## 3. P2 禁止事项

P2 阶段禁止：

1. 接入真实 AI。
2. 接入真实支付。
3. 创建完整账户系统。
4. 读取或处理全部古籍资料。
5. 编造古籍原文、书名或出处。
6. 输出绝对化断语。
7. 加入英文版、宝宝起名或分享返佣。
8. 承诺祈福一定灵验、改命、治病、驱邪、法术或官方寺庙保证。

## 4. 古籍内容处理规则

真实古籍内容由用户通过其他方式收集。

P2 只允许使用占位文案：

> 真实古籍出处将在 P3 古籍知识库阶段接入。当前为 mock 展示，不作为正式古籍结论。

当 P3 开始或 P2 需要真实片段时，Codex 必须向用户索取古籍内容。

## 5. P2 验收口径

P2 完成时应满足：

- 本地可运行。
- 页面可点击。
- 所有 MVP 功能入口存在。
- 所有表单可进入 mock 报告。
- 所有报告包含古籍出处占位和风险提醒。
- mock 支付流程可走通但不真实扣款。
- 我的记录使用 localStorage。
- 移动端基础适配可用。
