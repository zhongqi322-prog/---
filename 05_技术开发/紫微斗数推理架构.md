# 紫微斗数推理架构

## 当前目标

第一版只设计稳定接口与 mock 推理结构，不实现真实完整排盘算法。紫微斗数作为八字之外的付费增强内容。

## 输入

- 称呼
- 性别
- 出生日期
- 日期类型：公历、农历或不确定
- 是否闰月
- 出生时辰或未知
- 出生地点
- 问题类型
- 一句话困境描述

## 标准化

- 校验必填字段与日期格式。
- 将问题映射到相关宫位集合。
- 出生时辰未知时生成简版报告，并明确省略依赖时辰的内容。
- 第一版不得伪装为真实命盘计算结果。

## 问题映射

| 问题 | 关注宫位 |
|---|---|
| 姻缘感情 | 夫妻宫、命宫、福德宫、迁移宫 |
| 事业工作 | 官禄宫、命宫、财帛宫、迁移宫、交友宫 |
| 财运收入 | 财帛宫、官禄宫、田宅宫、福德宫 |
| 家庭房产 | 田宅宫、命宫、福德宫、迁移宫 |
| 家庭关系 | 父母宫、兄弟宫、命宫、福德宫 |
| 自我性格 | 命宫、身宫、福德宫、迁移宫 |
| 近期趋势 | 大限、流年、命宫、相关问题宫位 |

## 推理流水线

1. `input-normalizer`：标准化和校验输入。
2. `mock-chart`：生成明确标识为 mock 的命盘结构。
3. `palace-mapping`：将用户问题映射至宫位。
4. `mock-analyzer`：按固定规则生成分析要点。
5. `citation-selector`：选择已审核的古籍知识片段。
6. `report-builder`：组织统一报告并加入风险提醒。

## 报告结构

1. 简要结论
2. 命盘基础信息
3. 命宫与身宫解读
4. 当前问题相关宫位
5. 主星组合分析
6. 四化与三方四正
7. 大限与流年趋势
8. 古籍依据
9. 白话解释
10. 现实建议
11. 风险提醒

## 安全边界

- mock 结果必须明确标注。
- 不输出必然发生的事件。
- 不替代医疗、法律、投资或关系决策。
- 不知道出生时辰时，不分析依赖时辰才能确定的内容。
