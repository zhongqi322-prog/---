# P6报告保存与隐私边界

更新时间：2026-07-01

## 1. 目标

明确真实付费后报告如何保存、哪些字段可以保存、哪些字段必须最小化收集，以及用户如何查看、删除或联系客服处理。

本文件不是法律意见，正式上线前仍需按现行法律法规、支付平台规则和运营主体要求复核。

## 2. 数据分类

### 普通业务数据

- 服务类型。
- 订单号。
- 支付状态。
- 报告生成时间。
- 报告版本。
- 用户反馈。

### 敏感或高风险数据

- 出生日期、出生时间、出生地。
- 住址、户型、房屋照片。
- 手相图片。
- 梦境、姻缘、家庭关系、健康相关描述。
- 用户手机号、微信号、支付账户标识。

## 3. 最小保存策略

第一版建议：

- 订单数据：保存到满足售后、对账、退款和合规留存需要。
- 报告正文：默认保存，供用户回看；必须提供删除或联系客服删除路径。
- 原始表单：能不保存就不保存；如必须保存，只保存生成报告所需的结构化摘要。
- 图片类数据：第一版不建议真实上传保存；若后续必须上传，必须单独设计存储、访问权限和删除机制。
- 住址类数据：不保存精确门牌；住宅风水只保留用户自填的结构化空间描述。

## 3.1 字段级保留周期

以下为第一版产品设计建议，正式上线前仍需结合支付平台、税务、会计和隐私合规要求复核。

| 数据 | 是否保存 | 建议保留周期 | 删除方式 | 删除后保留内容 |
|---|---|---|---|---|
| 订单号、金额、支付状态 | 是 | 5 年或按财务/支付平台要求 | 不支持用户直接删除 | 保留对账所需字段 |
| 平台交易号 | 是 | 5 年或按支付平台要求 | 不支持用户直接删除 | 保留脱敏交易号 |
| 报告正文 | 是 | 默认 180 天，可续期 | 用户自助删除或客服删除 | 保留 `report_id`、订单关联和删除时间 |
| 表单原始输入 | 不建议保存 | 生成报告后立即丢弃 | 自动处理 | 只保留结构化摘要 |
| 结构化输入摘要 | 是 | 180 天 | 随报告删除 | 保留不可逆摘要或空值 |
| 出生日期、出生时间 | 尽量摘要化保存 | 180 天 | 随报告删除 | 保留是否完整填写的布尔值 |
| 出生地 | 仅保存城市级或更粗粒度 | 180 天 | 随报告删除 | 不保留精确地址 |
| 住宅风水地址 | 不保存精确地址 | 不保存 | 不适用 | 只保留空间描述摘要 |
| 手相图片、房屋照片 | 第一版不保存 | 不保存 | 不适用 | 不保留 |
| 手机号 / 联系方式 | 仅客服需要时保存 | 客服问题关闭后 180 天 | 客服删除 | 保留脱敏联系方式 |
| 模型输入输出日志 | 可选 | 30 天 | 到期自动删除 | 保留模板版本和风险规则版本 |

## 3.2 用户删除与导出

第一版最小能力：

- 用户可在“我的记录”中删除报告。
- 删除报告时同时删除报告正文、结构化输入摘要和可删除的敏感字段。
- 订单对账字段不随报告删除，但必须与报告正文解除展示关联。
- 用户可复制或下载报告文本；PDF 导出可作为后续增强，不作为第一版必需。
- 客服可按订单号协助删除报告，但不得删除支付对账所需字段。

删除状态建议：

- `active`：正常可查看。
- `delete_requested`：用户已申请删除，等待处理。
- `deleted`：正文和敏感摘要已删除。
- `retained_for_accounting`：仅保留订单和对账字段。

## 3.3 用户同意机制

付费前必须展示并要求用户确认：

- 报告会保存到“我的记录”。
- 哪些输入会被用于生成报告。
- 哪些内容会被保存，哪些内容不会保存。
- 用户可删除报告正文，但订单和支付对账字段可能按要求保留。
- 本服务为传统文化参考，不承诺现实结果。

## 4. 报告记录字段

最小报告字段：

- `report_id`
- `order_id`
- `service_slug`
- `user_id` 或匿名会话标识
- `input_summary`
- `report_summary`
- `report_body`
- `citations`
- `risk_reminder`
- `model_version` 或 `template_version`
- `created_at`
- `deleted_at`

## 5. 用户可见能力

第一版至少提供：

- 我的记录查看已生成报告。
- 付费订单状态说明。
- 删除报告说明或联系客服删除入口。
- 隐私说明入口。
- 风险提醒固定展示。

## 6. 不得保存或展示

- 不保存无关身份证件信息。
- 不保存银行卡号。
- 不保存支付密码或验证码。
- 不展示完整支付账户标识。
- 不把用户隐私内容作为公开案例，除非后续另行取得明确授权。

## 7. AI 调用前隐私处理

真实接入大模型前必须做：

- 只发送生成报告所需字段。
- 不发送支付信息。
- 不发送可直接识别用户身份的字段，除非必要且已告知。
- 对出生地、住址、图片等高风险字段做最小化处理。
- 保存模型输入输出日志前必须定义保留时间和访问权限。

## 8. 开发前验收

- 用户输入字段清单已确认。
- 每个字段是否保存、保存多久、是否可删除已确认。
- 我的记录与订单记录的关联方式已确认。
- 隐私说明、用户同意机制和删除路径已确认。
- 报告导出方式已确认：第一版文本复制 / 后续 PDF。
