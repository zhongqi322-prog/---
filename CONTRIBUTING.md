# 贡献指南

感谢参与传统术数古籍 AI 解读平台。

## 开始前

所有贡献者和 Codex Agent 必须先读取：

1. `00_项目总纲/PROJECT_STAGE.md`
2. `00_项目总纲/CONTEXT.md`
3. `00_项目总纲/ACTIVE_TASKS.md`
4. 当前阶段交接文
5. 与任务直接相关的文件

禁止无目的扫描或修改全项目。

## 工作原则

- 只处理当前阶段和任务范围内的内容。
- 不删除已确定的 MVP 功能。
- 不加入英文版、宝宝起名或分享返佣。
- 不擅自接入真实 AI、真实支付或真实寺庙服务。
- 不编造古籍原文、出处或确定性结论。
- 所有报告设计必须保留古籍出处和风险提醒。

## 古籍与数据贡献

- 录入前阅读 `03_古籍知识库/古籍录入规范.md` 和 `03_古籍知识库/出处引用规范.md`。
- 必须记录书名、版本、章节、来源、版权状态和审核状态。
- 原文、白话解释和模型生成内容必须分开。
- 版权状态未确认的资料不得进入正式产品知识库。

## 提交变更

1. 从最新 `main` 分支开始。
2. 保持变更范围单一、清晰。
3. 完成必要检查。
4. 更新相关任务或阶段文档。
5. 提交 Pull Request，并说明变更、验证方法和剩余风险。

## 提交信息建议

```text
docs: 更新产品流程
feat: 添加首页 mock 页面
fix: 修复报告引用显示
data: 添加已审核古籍片段
chore: 更新仓库配置
```
