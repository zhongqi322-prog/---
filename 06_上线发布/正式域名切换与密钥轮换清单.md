## 3. 正式 `www` 域名切换

目标：让 `https://www.laozuzongxuanxue.cn` 指向 CloudBase 云托管服务端环境，而不是旧静态托管源站。

腾讯云里只选择一种方式：

1. 推荐方式：在 CloudBase 云托管服务里绑定自定义域名 `www.laozuzongxuanxue.cn`。
2. 备选方式：在 CDN / EdgeOne 中把 `www` 的源站从旧静态源站切换到云托管服务对应源站。

切换时必须确认：

- [ ] HTTPS 证书已绑定 `www.laozuzongxuanxue.cn`。
- [ ] `www` 的访问路径 `/api/reports` 会到达云托管服务。
- [ ] 不修改 DeepSeek API Key 到前端代码。
- [ ] 不启用真实支付、订单、退款或真实扣款。

## 4. 切换后验收

正式域名切换后，在本地执行：

```powershell
npm run test:ai -- https://www.laozuzongxuanxue.cn
npm run test:p6 -- https://www.laozuzongxuanxue.cn
```

验收通过标准：

- [ ] `/api/reports` 返回结构化 AI 报告。
- [ ] 首页、功能页、填表、生成报告、模拟解锁、我的记录流程通过。
- [ ] 风险提醒和古籍出处可见。
- [ ] 模拟解锁仍不会跳转微信支付、不会输入付款密码、不会扣款、不会产生真实订单。
