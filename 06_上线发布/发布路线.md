# 发布路线

## 发布目标

第一版最终通过 GitHub 管理代码与文档，并发布到 Vercel，达到可访问、可点击、可填写、可生成 mock 报告、可展示古籍出处和风险说明的状态。

## 阶段路线

### 第 0 阶段：规则与文档

- 创建完整目录。
- 明确 MVP、任务范围、古籍规范和风险边界。
- 不编写网站代码。

### 第 1 阶段：可点击原型

- 创建 Next.js、React、TypeScript、Tailwind CSS 项目。
- 实现全部入口和移动端页面。
- 使用本地 JSON mock 数据。
- 实现 mock 报告、mock 支付与 localStorage 记录。

### 第 2 阶段：规则接口

- 补充八字、紫微、黄历、易经与风水的规则接口。
- 继续使用可验证的 mock 或确定性规则。

### 第 3 阶段：发布测试

- 创建 GitHub 仓库并检查敏感信息。
- 连接 Vercel 并部署预览环境。
- 完成移动端、链接、mock 数据、出处、风险话术检查。
- 通过验收后发布第一版。

## 上线检查重点

- 首页和所有入口可打开。
- 表单可提交，mock 报告可生成。
- 报告可显示古籍出处。
- mock 支付清晰标识。
- 我的记录可在本地保存和找回。
- 风险说明可见。
- 无绝对化断言、违规承诺或高风险建议。
- 手机端可正常访问。
