# 腾讯云国内部署指南

更新时间：2026-07-09

## 1. 当前前置条件

- `laozuzongxuanxue.cn` ICP 备案管局审核已通过。
- 当前网站已完成 DeepSeek AI-only 服务端接口本地接入和第一轮输出安全加固。
- 当前仍不接真实支付、不接订单退款结算、不接完整账户系统。
- 国内部署目标是提升中国境内手机访问稳定性，不承诺所有网络环境 100% 可访问。

## 2. 推荐部署方式

当前项目已经新增 `/api/reports` 服务端动态路由。

因此部署分为两种模式：

1. 静态托管模式：只能运行前端页面和本地测试报告兜底，不能运行真实 AI 接口。
2. 服务端模式：可运行 `/api/reports` 并调用 DeepSeek，必须配置服务端环境变量。

### 2.1 真实 AI 推荐方案

如要上线真实 AI，必须选择支持 Next.js 服务端 API / 云函数 / 云托管的方案。

推荐要求：

- 支持 `next build`。
- 支持 `next start` 或等效 Next.js 服务端运行时。
- 可配置服务端环境变量：

```text
DEEPSEEK_API_KEY=线上有效密钥
DEEPSEEK_MODEL=deepseek-v4-pro
```

构建命令：

```powershell
npm ci
npm run build
```

启动命令：

```powershell
npm run start
```

验收命令：

```powershell
npm run test:ai -- https://你的线上域名
npm run test:p6 -- https://你的线上域名
```

注意：

- 不得把 `.env.local` 上传到代码仓库。
- 不得把 `DEEPSEEK_API_KEY` 写入前端代码。
- 如果部署平台只支持静态托管，则真实 AI 不会生效。

### 2.1.1 CloudBase 云托管 Dockerfile 方式

当前 CloudBase 控制台的 Git 平台部署要求仓库内提供 `Dockerfile`。

仓库根目录已新增：

```text
Dockerfile
.dockerignore
```

控制台填写建议：

```text
目标目录：留空；如果控制台强制要求，填 .
Dockerfile 文件：有
Dockerfile 名称：Dockerfile
访问端口：80
服务端口：3000
服务名称：fengshui-ai-site
Git 分支：main
```

环境变量：

```text
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_API_KEY=线上新生成的有效密钥
```

注意：

- 正式部署前需要把 `Dockerfile`、`.dockerignore` 和当前代码提交并推送到 GitHub `main`。
- `DEEPSEEK_API_KEY` 建议在 DeepSeek 控制台重新生成后，直接填入腾讯云环境变量，不要写入代码或文档。
- 如果 GitHub `main` 还不是最新代码，先关闭自动部署，等推送完成后再开启。

### 2.2 静态兜底方案

优先方案：

1. 腾讯云 EdgeOne Pages 或 CloudBase 静态托管。
2. GitHub 仓库作为代码源。
3. 构建命令使用：

```powershell
npm ci
npm run build:static
```

4. 输出目录填写：

```text
out
```

该方案只适合作为无真实 AI 的静态兜底。页面表单会调用 `/api/reports` 失败后回退为本地测试报告。

如后续接入真实支付回调、订单系统、大模型 API 或账户系统，必须迁移到 CloudBase 云托管 / 云函数 / 云开发后端或其他支持服务端 API 的平台。

手动上传兜底方案：

- 本地执行 `npm run build:static` 后，使用 `out/` 目录作为静态站点内容。
- 当前已生成 `cloudbase-static-upload.zip`，压缩包根层级即 `out/` 内文件，可用于 CloudBase / 静态托管手动上传测试。
- 手动上传只作为部署验证或临时兜底；正式持续发布仍优先连接 GitHub `main` 并配置自动构建。

## 3. 本地静态构建验证

服务端部署前在本地执行：

```powershell
npm run lint
npm run build
npm run test:ai -- http://127.0.0.1:3008
npm run test:p6 -- http://127.0.0.1:3008
```

验收点：

- `npm run lint` 通过。
- `npm run build` 通过。
- `npm run test:ai` 通过，确认真实 AI 接口、古籍引用、风险提醒和高风险词检查通过。
- `npm run test:p6` 通过，确认页面完整流程通过。

静态兜底包另行执行：

```powershell
npm run build:static
```

验收点：

- `npm run build:static` 通过，并生成静态页面。
- 不把静态包当作真实 AI 上线方案。

## 4. 腾讯云控制台操作

1. 如果上线真实 AI，创建支持服务端 API 的云托管 / 云函数 / Next.js 服务端项目。
2. 如果只做静态兜底，创建 EdgeOne Pages / CloudBase 静态托管项目。
3. 连接 GitHub 仓库：`zhongqi322-prog/---`。
4. 选择 `main` 分支。
5. 服务端模式配置构建命令：`npm ci && npm run build`。
6. 服务端模式配置启动命令：`npm run start`。
7. 服务端模式配置环境变量：`DEEPSEEK_API_KEY`、`DEEPSEEK_MODEL`。
8. 静态模式配置构建命令：`npm ci && npm run build:static`，输出目录：`out`。
9. 等待首次部署完成。
10. 使用腾讯云提供的临时访问地址做 smoke test。

## 5. 域名绑定

首次部署验证通过后，再绑定：

- `laozuzongxuanxue.cn`
- `www.laozuzongxuanxue.cn`

必须开启 HTTPS。

DNS 解析建议：

- 根域名按腾讯云控制台要求配置 CNAME / A / ALIAS。
- `www` 子域名按腾讯云控制台要求配置 CNAME。
- 不要同时保留冲突解析记录。

## 6. 上线后检查

绑定域名和 HTTPS 生效后，执行：

```powershell
python scripts/p5_public_smoke_test.py https://laozuzongxuanxue.cn
python scripts/p5_public_smoke_test.py https://www.laozuzongxuanxue.cn
npm run test:ai -- https://www.laozuzongxuanxue.cn
npm run test:p6 -- https://www.laozuzongxuanxue.cn
```

同时用手机流量和常用 Wi-Fi 实测：

- 首页是否能打开。
- 服务页是否能打开。
- 表单是否能生成 AI 文化参考报告；如果是静态兜底模式，确认能回退本地测试报告。
- mock 支付弹窗是否明确不扣款。
- 风险说明和古籍出处是否可见。
