# 腾讯云国内部署指南

更新时间：2026-06-30

## 1. 当前前置条件

- `laozuzongxuanxue.cn` ICP 备案管局审核已通过。
- 当前网站仍是 MVP mock 原型，不接真实 AI、不接真实支付、不接完整账户系统。
- 国内部署目标是提升中国境内手机访问稳定性，不承诺所有网络环境 100% 可访问。

## 2. 推荐部署方式

当前项目没有 API Route、SSR 数据依赖或服务端存储，适合先走静态站点部署。

优先方案：

1. 腾讯云 EdgeOne Pages 或 CloudBase 静态托管。
2. GitHub 仓库作为代码源。
3. 构建命令使用：

```powershell
npm ci
npm run build:static
```

4. 输出目录填写：

```text
out
```

如后续接入真实支付回调、订单系统、大模型 API 或账户系统，再迁移到 CloudBase 云托管 / 云函数 / 云开发后端。

## 3. 本地静态构建验证

部署前在本地执行：

```powershell
npm run lint
npm run build
npm run build:static
```

验收点：

- `npm run lint` 通过。
- `npm run build` 通过，确保现有 Vercel / Next 构建路径不被破坏。
- `npm run build:static` 通过，并生成 `out/` 目录。

## 4. 腾讯云控制台操作

1. 创建 EdgeOne Pages / CloudBase 静态托管项目。
2. 连接 GitHub 仓库：`zhongqi322-prog/---`。
3. 选择 `main` 分支。
4. 配置构建命令：`npm ci && npm run build:static`。
5. 配置输出目录：`out`。
6. 等待首次部署完成。
7. 使用腾讯云提供的临时访问地址做 smoke test。

## 5. 域名绑定

首次部署验证通过后，再绑定：

- `laozuzongxuanxue.cn`
- `www.laozuzongxuanxue.cn`

必须开启 HTTPS。

DNS 解析建议：

- 根域名按腾讯云控制台要求配置 CNAME / A / ALIAS。
- `www` 子域名按腾讯云控制台要求配置 CNAME。
- 不要同时保留冲突解析记录。

## 6. 上线后检查

绑定域名和 HTTPS 生效后，执行：

```powershell
python scripts/p5_public_smoke_test.py https://laozuzongxuanxue.cn
python scripts/p5_public_smoke_test.py https://www.laozuzongxuanxue.cn
```

同时用手机流量和常用 Wi-Fi 实测：

- 首页是否能打开。
- 服务页是否能打开。
- 表单是否能生成 mock 报告。
- mock 支付弹窗是否明确不扣款。
- 风险说明和古籍出处是否可见。
