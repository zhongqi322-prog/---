# P5发布前Smoke Test规则

## 1. 目的

固化 P2 审核建议优化项：P5 发布前必须再次使用 Browser 或 Playwright 做公开地址 smoke test。

## 2. 触发时机

以下情况必须执行：

1. 部署到 Vercel 后。
2. 修改首页、功能入口、报告、支付弹窗、记录页后。
3. 发布正式用户测试链接前。

## 3. 推荐工具

优先顺序：

1. Browser 插件：检查公开地址、移动端视觉、点击路径。
2. `webapp-testing` + Playwright：自动化检查核心路径。
3. HTTP 路由检查：作为 Browser 不稳定时的补充。

## 4. 必测路径

1. 首页公开地址返回 200。
2. 首页今日黄历卡片可见。
3. 9 个核心服务入口可点击。
4. 八字解读表单可提交。
5. 紫微斗数付费报告入口可访问。
6. mock 支付弹窗可打开，且明确不真实扣款。
7. 古籍出处占位可见。
8. 风险提醒可见。
9. 我的记录 localStorage 流程可用。
10. 移动端 390px 宽度可基本使用。

## 5. 禁止事项

P5 smoke test 不得：

1. 接入真实支付。
2. 接入真实 AI。
3. 编造古籍原文或出处。
4. 修改真实用户数据。
5. 承诺祈福现实结果。

## 6. 输出要求

P5 必须输出：

- smoke test 记录
- 失败项列表
- 修复记录
- 是否建议公开给用户测试
