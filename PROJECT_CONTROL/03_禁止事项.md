# 03_禁止事项

更新时间：2026-07-09

## 1. 全局禁止

1. 禁止无任务目标地扫描全项目。
2. 禁止无目的读取全部历史对话。
3. 禁止修改与当前任务无关的文件。
4. 禁止删除已确定 MVP 功能。
5. 禁止创建第二套知识库或第二套项目结构。
6. 禁止把 Obsidian 当作独立事实源。

## 2. 产品范围禁止

1. 禁止重新讨论英文版，除非用户明确要求。
2. 禁止重新加入宝宝起名。
3. 禁止加入分享返佣。
4. 禁止把今日黄历做成独立主项目。
5. 禁止擅自删除八字、紫微斗数、姻缘解惑、住宅风水、易经问事、祈福心愿、周公解梦、手相解析、古籍问答、我的记录、价格解锁、风险说明。

## 3. AI 与支付禁止

1. 除已授权的 PC-0009 / PC-0010 DeepSeek AI-only 服务端接口外，禁止新增其他真实大模型 API。
2. 当前禁止接入真实支付。
3. 当前禁止创建完整账户系统。
4. 禁止把 mock 支付包装成真实支付。
5. 禁止绕过 P6 前置设计直接开发真实收款。
6. 禁止在未设计内容安全规则前让模型自由生成命理结论。
7. 禁止在未复验支付回调、订单状态和退款链路前上线真实收款。
8. 禁止在未明确报告保存和删除路径前保存用户敏感资料。
9. 禁止把“进入下一阶段”理解为可以跳过小范围用户复测、支付资质确认和退款权限验证直接上线真实收款。
10. 禁止让模型自行声称完成真实八字排盘、紫微排盘、易经起卦、手相图片识别、梦境预兆断定或风水吉凶断定。
11. 禁止把纯静态托管写成真实 AI 已上线；只有 `/api/reports` 在线上返回有效 API 响应并通过 `npm run test:ai` 后，才可记录为真实 AI 线上可用。

## 4. 古籍内容禁止

1. 禁止编造古籍原文。
2. 禁止编造书名、章节、作者、版本、出处。
3. 禁止使用未确认来源的古籍内容作为正式产品依据。
4. 禁止无出处输出结论。
5. 禁止读取全部 raw 古籍文件，除非任务明确需要。

## 5. 命理与风控禁止

1. 禁止输出绝对化断语。
2. 禁止恐吓用户。
3. 禁止承诺一定灵验。
4. 禁止承诺保证实现。
5. 禁止承诺改命。
6. 禁止承诺消灾必成。
7. 禁止提供治病承诺。
8. 禁止提供驱邪、法术、官方寺庙保证。
9. 禁止替代医疗、法律、投资、心理咨询等专业建议。

## 6. 祈福心愿模块禁止

1. 禁止承诺心愿一定实现。
2. 禁止承诺祈福一定有效。
3. 禁止暗示官方寺庙背书，除非未来有真实合规合作并经用户明确确认。
4. 禁止把 mock 线下祈愿流程写成真实履约。
5. 禁止在未建立线下履约记录、授权和回传机制前，把“寺庙祈福点灯”写成已经发生的真实服务。

## 7. 国内部署禁止

1. 禁止在腾讯云部署和临时地址 smoke test 未通过前，直接把 `laozuzongxuanxue.cn` 正式解析到线上服务。
2. 禁止承诺中国大陆所有网络 100% 可访问。
3. 禁止忽略 HTTPS。
4. 正式上线后禁止忽略公安联网备案提醒。
5. 禁止把当前环境尚未复验通过的 HTTPS / `www` 访问写成已通过。

## 8. 首页与文案禁止

1. 禁止把 mock 黄历写成真实黄历。
2. 禁止使用“趋吉避凶”“运势吉凶推断”“揭示潜意识”“性格运势解读”等偏绝对或过度营销表达作为核心承诺。
3. 禁止为了视觉冲击牺牲风险边界说明。
4. 禁止让黑金八卦风格变成恐吓、迷信或博彩感。

## 9. 工程操作禁止

1. 禁止执行破坏性 git 操作，例如 `git reset --hard` 或随意回滚用户改动。
2. 禁止提交或推送前不说明变更范围。
3. 禁止跳过 lint/build 验证后声称已完成前端改动。
4. 禁止在工作区已有无关改动时擅自覆盖。
