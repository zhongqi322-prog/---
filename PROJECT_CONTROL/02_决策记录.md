# 02_决策记录

更新时间：2026-07-01

## 1. 项目系统决策

1. Obsidian、GitHub、Codex 必须视为同一个项目系统。
2. GitHub 仓库是唯一真实数据源。
3. Obsidian 是项目大脑和编辑界面，不是第二套知识库。
4. Codex 是执行开发者，不维护独立知识体系。
5. 后续新会话优先读取 `PROJECT_CONTROL/` 下 5 个控制文件。

## 2. 产品范围决策

1. 当前只做中文版 MVP。
2. 英文版暂停，等中文版盈利后再考虑。
3. 不做宝宝起名，改为姻缘解惑。
4. 暂不做分享返佣。
5. 今日黄历放在首页，不作为独立主项目。
6. 紫微斗数保留，并作为付费增强内容。
7. 祈福心愿、周公解梦、手相解析保留。

## 3. MVP 功能范围

第一版必须保留：

1. 首页今日黄历卡片
2. 八字解读
3. 紫微斗数付费报告
4. 姻缘解惑
5. 住宅风水
6. 易经问事
7. 祈福心愿
8. 周公解梦
9. 手相解析
10. 古籍问答
11. 我的记录轻量版
12. 价格 / 解锁完整报告
13. 风险说明

不得擅自删除以上功能。

## 4. 技术路线决策

1. 当前网站使用 Next.js + React + TypeScript + Tailwind CSS。
2. 第一版使用 mock 数据。
3. 第一版不接真实 AI。
4. 第一版不接真实支付。
5. 第一版不做完整账户系统。
6. 第一版使用 localStorage 做轻量记录。
7. Vercel 已作为海外/临时公开部署方案验证过。
8. 中国境内访问稳定性不足，国内正式访问改走腾讯云方案。

## 5. 国内上线决策

1. 域名 `laozuzongxuanxue.cn` 已在腾讯云购买。
2. ICP 备案管局审核已通过。
3. 备案通过后允许启动国内域名绑定和解析，但必须先完成腾讯云部署验证。
4. 国内部署优先使用腾讯云静态托管 / EdgeOne Pages / CloudBase；后续如接入真实支付、订单、大模型 API，再迁移到云托管 / 云函数后端。
5. 绑定域名后需要开启 HTTPS。
6. 网站正式开通后需要关注公安联网备案。
7. 2026-07-01 用户确认线上部署已完成。
8. 当前环境验证 `http://laozuzongxuanxue.cn/` 返回 200；HTTPS 与 `www` 访问仍需复验，不得写成已完全通过。

## 6. 古籍与内容决策

1. 古籍内容由用户外部收集，Codex 在需要阶段向用户索取。
2. Codex 不得自行编造古籍原文、书名、章节或出处。
3. 所有正式报告必须包含古籍出处模块。
4. 古籍出处模块必须包含书名、原文片段、白话解释、与用户问题的关系、风险提醒。
5. 当前已整理过部分古籍结构化内容，包括周易、子平真诠、周公解梦等。
6. 用户确认过部分来源和可用范围，但正式商业使用仍需谨慎保留版权/版本记录。

## 7. 风险与合规决策

1. 所有解读只作为传统文化参考和自我整理。
2. 不承诺现实结果。
3. 不输出绝对化预测。
4. 不恐吓用户。
5. 不替代医疗、法律、投资、心理咨询等专业建议。
6. 祈福心愿模块只能做心愿记录、祝福表达和 mock 展示，不承诺灵验。

## 8. 大模型 API 决策

1. 当前 P6 前置设计阶段不接入大模型 API。
2. 大模型 API 应在 P6 设计通过并完成小范围用户测试后再接入。
3. 接入前必须先完成真实支付方案、用户付费验证、隐私边界、内容安全和报告生成规则设计。
4. 大模型输出必须受古籍出处、结构化规则、风险边界约束。

## 10. P6 真实付费与运营决策

1. P6 已启动，但先做前置设计，不直接开发真实支付或真实 AI。
2. 真实支付必须先有订单号、订单状态机、支付回调验签、失败取消超时和退款处理。
3. 报告保存必须先定义保存字段、保留时间、删除路径和敏感信息边界。
4. 真实 AI 必须先定义结构化输出、古籍引用来源、风险话术和内容安全拦截规则。
5. 小范围用户测试反馈不足前，不进入真实收款开发。

## 9. 当前首页视觉决策

1. 首页已按用户提供参考图改为黑金八卦 + 古籍卡片风格。
2. 视觉方向强调古籍、八卦、罗盘、黑金、高端文化产品感。
3. 文案必须保持文化参考语气，避免“保证、改命、趋吉避凶、绝对预测”等表达。
4. 今日黄历当前必须标识为 mock，不得伪装成真实黄历。
5. 2026-06-25 用户要求将本项目所有页面按参考图风格调整，当前代码已扩展到首页、服务页、记录页、表单、报告、支付弹窗、古籍出处和风险提醒组件。
6. 全站视觉统一不改变 MVP 功能范围，不接真实 AI，不接真实支付。
