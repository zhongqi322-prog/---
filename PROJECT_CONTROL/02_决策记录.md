# 02_决策记录

更新时间：2026-07-09

## 1. 项目系统决策

1. Obsidian、GitHub、Codex 必须视为同一个项目系统。
2. GitHub 仓库是唯一真实数据源。
3. Obsidian 是项目大脑和编辑界面，不是第二套知识库。
4. Codex 是执行开发者，不维护独立知识体系。
5. 后续新会话优先读取 `PROJECT_CONTROL/` 下 5 个控制文件。

## 2. 产品范围决策

1. 当前只做中文版 MVP。
2. 英文版暂停，等中文版盈利后再考虑。
3. 不做宝宝起名，改为姻缘解惑。
4. 暂不做分享返佣。
5. 今日黄历放在首页，不作为独立主项目。
6. 紫微斗数保留，并作为付费增强内容。
7. 祈福心愿、周公解梦、手相解析保留。

## 3. MVP 功能范围

第一版必须保留：

1. 首页今日黄历卡片
2. 八字解读
3. 紫微斗数付费报告
4. 姻缘解惑
5. 住宅风水
6. 易经问事
7. 祈福心愿
8. 周公解梦
9. 手相解析
10. 古籍问答
11. 我的记录轻量版
12. 价格 / 解锁完整报告
13. 风险说明

不得擅自删除以上功能。

## 4. 技术路线决策

1. 当前网站使用 Next.js + React + TypeScript + Tailwind CSS。
2. 第一版使用 mock 数据。
3. 第一版不接真实 AI。
4. 第一版不接真实支付。
5. 第一版不做完整账户系统。
6. 第一版使用 localStorage 做轻量记录。
7. Vercel 已作为海外/临时公开部署方案验证过。
8. 中国境内访问稳定性不足，国内正式访问改走腾讯云方案。

## 5. 国内上线决策

1. 域名 `laozuzongxuanxue.cn` 已在腾讯云购买。
2. ICP 备案管局审核已通过。
3. 备案通过后允许启动国内域名绑定和解析，但必须先完成腾讯云部署验证。
4. 国内部署优先使用腾讯云静态托管 / EdgeOne Pages / CloudBase；后续如接入真实支付、订单、大模型 API，再迁移到云托管 / 云函数后端。
5. 绑定域名后需要开启 HTTPS。
6. 网站正式开通后需要关注公安联网备案。
7. 2026-07-01 用户确认线上部署已完成。
8. 当前环境验证 `http://laozuzongxuanxue.cn/` 返回 200；`http://www.laozuzongxuanxue.cn/` 可跳转到 `https://www.laozuzongxuanxue.cn/` 且最终返回 200；`https://www.laozuzongxuanxue.cn/` 已复验通过；根域名 `https://laozuzongxuanxue.cn/` 仍返回 CDN 514，不得写成根域名 HTTPS 已完全通过。

## 6. 古籍与内容决策

1. 古籍内容由用户外部收集，Codex 在需要阶段向用户索取。
2. Codex 不得自行编造古籍原文、书名、章节或出处。
3. 所有正式报告必须包含古籍出处模块。
4. 古籍出处模块必须包含书名、原文片段、白话解释、与用户问题的关系、风险提醒。
5. 当前已整理过部分古籍结构化内容，包括周易、子平真诠、周公解梦等。
6. 用户确认过部分来源和可用范围，但正式商业使用仍需谨慎保留版权/版本记录。

## 7. 风险与合规决策

1. 所有解读只作为传统文化参考和自我整理。
2. 不承诺现实结果。
3. 不输出绝对化预测。
4. 不恐吓用户。
5. 不替代医疗、法律、投资、心理咨询等专业建议。
6. 祈福心愿模块只能做心愿记录、祝福表达和 mock 展示，不承诺灵验。
7. 祈福心愿可在正式运营设计中表达“线下寺庙祈福、点灯、拍照或视频回传记录”，但不得表达已经真实履约、官方寺庙背书或保证灵验。

## 8. 大模型 API 决策

1. P6 前置设计阶段不接入大模型 API 的限制已在小范围用户复测准入后解除。
2. 2026-07-08 用户确认先跳过支付阶段，优先接入 AI 大模型推算；支付以后再补充。
3. 第一版 AI-only 接入使用 DeepSeek `deepseek-v4-pro`，服务端通过 `DEEPSEEK_API_KEY` 调用，不得把密钥放到前端代码。
4. 大模型输出必须受古籍出处、结构化规则、风险边界约束。
5. 当前仍不得接真实支付、订单、退款、结算或真实扣款。
6. 2026-07-09 第一轮真实 AI 输出审核发现模型自行写出八字四柱风险，已决定禁止模型自行声称完成真实排盘、起卦、图像识别、梦境预兆断定或风水吉凶断定。
7. 2026-07-09 真实 AI 线上启用必须使用支持 Next.js 服务端 API / 云函数 / 云托管的部署环境；纯静态托管只能作为无真实 AI 的兜底页面方案。

## 10. P6 真实付费与运营决策

1. P6 已启动，但先做前置设计，不直接开发真实支付或真实 AI。
2. 真实支付必须先有订单号、订单状态机、支付回调验签、失败取消超时和退款处理。
3. 报告保存必须先定义保存字段、保留时间、删除路径和敏感信息边界。
4. 真实 AI 必须先定义结构化输出、古籍引用来源、风险话术和内容安全拦截规则。
5. 小范围用户测试反馈不足前，不进入真实收款开发。
6. 第一版支付平台选用微信支付。
7. 测试阶段使用个人主体只做产品和技术测试，不真实收款。
8. 正式阶段升级个体工商户后再进入真实收款。
9. 退款权限正式阶段再开通；未开通退款权限前不得上线真实收款。
10. 先做完整可用产品，再做小范围用户测试。
11. 2026-07-01 用户要求进入下一阶段；当前下一阶段定义为“小范围用户复测与真实付费准入验证”，不是直接进入真实支付或真实 AI 开发。
12. 进入真实付费开发前，必须先满足小范围用户反馈最低信号，并确认微信支付商户号、签约产品、结算账户和退款权限。
13. 2026-07-08 小范围用户复测准入信号已满足；下一阶段定义为“真实支付 / 报告保存 / 真实 AI 开发前专项审核”，仍不是直接开发真实支付或真实 AI。专项审核通过前，不得把模拟解锁改成真实扣款。
14. 2026-07-08 用户确认先不做支付阶段，跳过支付阶段，直接开始用 AI 大模型根据用户填入资料进行推算；支付以后再补充。当前允许进入 AI-only 接入，但仍不得接真实支付、订单、退款、结算或真实扣款。
