# P4_推理与报告阶段_交接文

## 1. 本阶段目标

完成 mock 推理与报告生成逻辑，让用户提交表单后可以生成结构化 mock 报告，并统一接入已审核古籍出处模块和风险提醒。

## 2. 本阶段完成内容

- 建立 `lib/classics.ts`，统一读取已审核、可使用的古籍片段。
- 建立 `lib/mock-report.ts`，生成确定性 mock 报告，不调用真实 AI。
- 扩展 localStorage 记录结构，保存生成后的结构化报告。
- 表单提交后生成包含摘要、用户资料整理、问题重述、分析结果、现实建议、不建议事项、追问、古籍出处和风险提醒的报告。
- 报告卡已接入真实古籍出处展示，不再使用 P2 占位内容。
- mock 解锁后展示完整报告区域，仍不产生真实支付。
- 首页空状态仍兼容古籍出处组件。

## 5. 当前有效决策

- P4 只做 mock 推理，不接真实 AI。
- P4 不接真实支付。
- 报告只能引用 `copyright_status = 可使用` 且 `review_status = 已审核` 的古籍片段。
- 报告必须展示古籍出处、原文、白话解释、与用户问题的关系和风险提醒。
- 报告不得输出绝对化预测、现实结果承诺或恐吓性结论。
