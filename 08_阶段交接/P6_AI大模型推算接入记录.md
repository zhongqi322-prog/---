## 3. 当前安全边界

- API Key 只允许放在服务端环境变量，不得写入前端代码。
- AI 输入只发送用户填写资料、服务信息和已审核古籍片段。
- 不向 AI 发送支付信息、银行卡、验证码或无关身份信息。
- 古籍出处仍由代码从已审核结构化数据中注入，不允许模型自由编造。
- 风险提醒由代码固定注入。
- AI 输出仍只作为传统文化参考，不承诺现实结果。
- 支付按钮仍为模拟解锁说明，不会跳转微信支付，不会产生真实订单，不会扣款。

## 4. 环境要求

本功能需要服务端运行环境。

本地或部署环境需配置：

```env
DEEPSEEK_API_KEY=你的服务端密钥
DEEPSEEK_MODEL=deepseek-v4-pro
```

如果未配置 `DEEPSEEK_API_KEY`：

- 页面仍可生成本地测试报告。
- 用户会看到“AI 生成暂不可用，已改用本地测试报告”。
- 不影响表单、报告展示、模拟解锁和我的记录流程。

## 5. 部署影响

新增 `/api/reports` 为服务端动态路由。

因此：

- 普通 `npm run build` 可用。
- `next start` 或支持 Next.js 服务端函数的平台可运行 AI 接口。
- 纯静态导出 / 静态托管不能运行 AI 接口；若继续使用纯静态托管，只能走本地测试报告兜底。
- 后续国内部署如要启用真实 AI，需要迁移到支持云函数 / 云托管 / 服务端 API 的环境。
