# 系统一致性原则

本文件是项目最高优先级规则。若其他文档、工作方式或 Agent 指令与本文件冲突，以本文件为准。

## 三者统一原则

Obsidian、GitHub 与 Codex 必须操作同一个项目系统，不允许分裂为不同知识体系或不同项目结构。

唯一项目目录：

```text
D:\工作\2026\老祖宗-玄学\AI-Projects\fengshui-ai-site
```

## 三者职责

### Obsidian：项目大脑与编辑界面

- 用于阅读项目结构、编辑需求、维护产品设计、管理古籍知识和记录决策。
- Obsidian 必须直接打开唯一项目目录。
- Obsidian 不是独立数据源，不在其他 Vault 中维护本项目副本。

### GitHub：唯一真实数据源

- 所有代码、Markdown、JSON 和古籍结构化内容必须进入同一个 GitHub 仓库。
- 所有正式内容以 GitHub 默认分支中的已提交版本为准。
- 未提交、未推送或仓库外文件不属于正式项目事实。

### Codex：执行开发者

- Codex 只读取和修改唯一项目目录。
- Codex 不维护独立知识体系，不在其他目录建立项目副本。
- Codex 完成任务后必须运行检查，并更新任务看板和开发日志。

## 强制工作流

1. 从 GitHub 获取最新版本。
2. 在唯一项目目录中使用 Obsidian 阅读和编辑。
3. Codex 在同一目录中执行文档、代码和数据任务。
4. 检查变更后提交并推送 GitHub。
5. 以 GitHub 默认分支确认正式版本。

## 禁止事项

- 禁止在 `风水命理项目/`、`风水命理AI网站/` 或其他目录继续维护本项目正式内容。
- 禁止只在 Obsidian 中保留未进入仓库的正式知识。
- 禁止 Codex 在仓库外生成正式代码、Markdown、JSON 或古籍结构化内容。
- 禁止将未提交或未推送内容描述为 GitHub 正式版本。
- 禁止多个仓库同时声明为本项目事实源。
