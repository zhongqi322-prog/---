# MVP 范围

## 目标

做出一个可点击、可填写、可生成 mock 报告、可展示古籍出处与付费入口的中文移动端友好网站，并发布到 Vercel。

## 必须保留

| 模块 | MVP 交付 |
|---|---|
| 首页 | 品牌区、今日黄历卡片、服务入口、可信说明、风险说明 |
| 八字解读 | 表单、mock 结构化结果、古籍出处、报告展示 |
| 紫微斗数 | 用户表单、mock 命盘、付费增强报告展示 |
| 姻缘解惑 | 双方资料、问题选择、mock 报告 |
| 住宅风水 | 住宅资料、mock 规则分析、现实调整建议 |
| 易经问事 | 问题输入、mock 卦象、行动建议 |
| 祈福心愿 | 心愿记录、心愿灯、家人祈愿、回传说明、mock 记录 |
| 周公解梦 | 梦境与情绪输入、传统与现代解释 |
| 手相解析 | 图片上传 mock、隐私提示、mock 报告 |
| 古籍问答 | 问题输入、古籍原文、出处与白话解释 |
| 我的记录 | localStorage 最近记录、reportId、链接复制 |
| 价格与解锁 | mock 支付弹窗、免费摘要与完整报告区分 |
| 风险说明 | 传统文化参考、非绝对预测、专业建议边界 |

## 第一版不做

- 英文版
- 宝宝起名
- 分享返佣
- 真实 AI 接口与大规模 RAG
- 真实支付和订单系统
- 完整账户与多设备同步
- 真实图片识别
- 真实寺庙合作履约

## 报告统一要求

每份报告必须包含：

1. 简要结论
2. 用户资料整理
3. 分析结果
4. 古籍依据与原文片段
5. 白话解释
6. 与用户问题的关系
7. 现实建议
8. 不建议做的事
9. 风险提醒

## 验收标准

- 所有入口可点击并能完成基本流程。
- 所有报告均显示出处与风险提醒。
- mock 支付明确标识为模拟功能。
- 移动端可使用。
- 无绝对化预测、违规承诺或高风险专业建议。
