# 传统术数古籍 AI 解读平台｜Codex 项目启动总文档

> 本文档用于让 Codex 直接读取并启动项目。  
> 当前阶段只做中文版 MVP，英文版暂停。  
> 工具链只使用：Obsidian + GitHub + Codex 多 Agent。  
> 暂不使用 Claude Code / OpenClaw。  

---

## 0. 项目一句话定位

本项目是一个 **基于中国传统术数古籍、规则算法与 AI 报告生成的玄学咨询平台**。

核心目标不是让 AI 无依据地“算命”，而是：

> 用户提供资料后，系统通过八字、紫微斗数、风水、易经、解梦、手相等传统体系，结合古籍出处与结构化推理，生成有依据、有解释、有边界的解读报告。

## 2. 项目核心原则

### 2.1 必须强调古籍出处

本项目最大的差异化是：

> 不是 AI 无根据输出答案，而是基于古籍、规则和用户资料进行推测与解释。

每份报告必须包含：

```text
古籍依据
原文片段
白话解释
与用户问题的关系
现实建议
风险提醒
```

### 2.2 AI 的角色

AI 不是“神秘算命核心”，而是：

```text
用户问题理解
古籍资料检索后的解释
报告组织
白话翻译
语气控制
现实建议生成
风险边界提醒
```

底层计算应尽量由规则/算法完成，例如：

```text
八字排盘
五行统计
十神关系
紫微命盘结构
黄历数据
易经起卦
风水表单规则
```

第一版可以使用 mock 数据，但接口结构要按未来真实算法预留。

### 2.3 禁止内容

所有 Agent 不得输出：

```text
必定发财
必定复合
必定离婚
必有灾祸
一定灵验
保证改命
治病
驱邪
法术
符咒服务
修炼指导
投资建议
医疗建议
法律建议
恐吓式断语
诱导高额消费
```

### 2.4 可用表达

可以使用：

```text
传统文化参考
古籍推演
象征性解读
趋势提醒
现实建议
心愿记录
线下祈愿记录
照片 / 视频回传
不承诺现实结果
```

## 3. 功能范围

### 3.1 第一版保留的完整功能

第一版不删减功能，保留以下入口和模块：

```text
1. 首页今日黄历卡片
2. 八字解读
3. 紫微斗数付费报告
4. 姻缘解惑
5. 住宅风水
6. 易经问事
7. 祈福心愿
8. 周公解梦
9. 手相解析
10. 古籍问答
11. 我的记录轻量版
12. 价格 / 解锁完整报告
13. 风险说明
```

### 3.2 不做宝宝起名

原本的“宝宝起名”入口取消，改为：

```text
姻缘解惑
```

原因：姻缘感情需求更高频、更容易付费。

### 3.3 今日黄历不是独立主入口

今日黄历不作为单独功能项目，而是在首页显著位置展示。

首页今日黄历卡片包含：

```text
今日宜忌
今日五行
今日冲煞
今日吉时
今日提醒
```

它的作用是：

```text
增强每日复访
营造传统术数氛围
引导用户进入其他付费功能
```

首页应参考同类网站优点：入口清楚、仪式感强、付费转化自然、记录可找回。

```text
1. 顶部品牌区
   - 传统术数古籍 AI 解读平台
   - 基于古籍出处，为现代人生困惑提供参考

2. 今日黄历卡片
   - 今日宜忌
   - 今日五行
   - 今日冲煞
   - 今日吉时
   - 今日提醒

3. 核心服务入口
   - 八字解读
   - 姻缘解惑
   - 紫微斗数
   - 住宅风水
   - 易经问事
   - 祈福心愿
   - 周公解梦
   - 手相解析
   - 古籍问答

4. 为什么可信
   - 古籍出处
   - AI 辅助整理

5. 付费报告展示

   - 解锁完整报告

6. 我的记录
   - 本地记录
   - 报告链接
   - 订单号找回预留

7. 风险说明
   - 传统文化参考
   - 不作绝对预测
   - 不替代医疗、法律、投资建议
```


## 10. 紫微斗数模块

### 10.1 产品定位

紫微斗数保留，并作为付费报告的重要增强内容。

建议产品包装：

```text
免费 / 低价：
八字基础解读

付费增强：
八字 + 紫微斗数深度报告
```

紫微斗数也可以单独作为付费报告入口或在姻缘解惑、事业分析中作为高级内容展示。

### 10.2 紫微斗数和八字的区别

普通用户解释：

```text
八字更像看你的底层五行和人生趋势。
紫微斗数更像看你人生十二个领域的分布，例如感情、事业、财运、家宅、人际。
```

### 10.3 用户需要提供的信息

普通用户表单：

```text
1. 你的称呼
2. 性别
3. 出生日期
4. 你填写的是哪种生日？
   - 公历生日
   - 农历生日
   - 不确定
5. 如果是农历生日，是否为闰月？
   - 不是
   - 是
   - 不确定
6. 出生时间
   - 不知道具体时间
   - 23:00-00:59 子时
   - 01:00-02:59 丑时
   - 03:00-04:59 寅时
   - 05:00-06:59 卯时
   - 07:00-08:59 辰时
   - 09:00-10:59 巳时
   - 11:00-12:59 午时
   - 13:00-14:59 未时
   - 15:00-16:59 申时
   - 17:00-18:59 酉时
   - 19:00-20:59 戌时
   - 21:00-22:59 亥时
7. 出生地点
   - 国家 / 省份 / 城市
8. 你这次主要想问什么？
   - 姻缘感情
   - 事业工作
   - 财运收入
   - 家宅房产
   - 家庭关系
   - 近期运势
   - 自我性格
   - 其他
9. 请用一句话描述你的困惑
```

### 10.4 用户看不到但后台需要转换的信息

## 17. 古籍录入与知识库

### 17.1 古籍放在哪里

古籍不要分散在不同平台。

正确方式：

```text
项目仓库 = 唯一资料库
Obsidian = 打开和编辑项目仓库的工具
GitHub = 同步和版本管理
Codex = 读取并处理同一套文件
```

### 17.2 古籍三层结构

#### 第一层：原始资料层
